
============================
How to Use 100-matrix_mul.py
============================

This module defines a matrix multiplication function ``matrix_mul(mtx_a, mtx_b)``.

Usage
=====
``matrix_mul(...)`` returns a new matrix representing the multiplication of
``mtx_a`` by ``mtx_b``.

::

    >>> matrix_mul = __import__('100-matrix_mul').matrix_mul
    >>> mtx_a = [
    ... [1, 2],
    ... [3, 4],
    ... ]
    >>> mtx_b = mtx_a
    >>> print(matrix_mul(mtx_a, mtx_b))
    [[7, 10], [15, 22]]

::

    >>> mtx_a = [[1, 2]]
    >>> mtx_b = [
    ... [3, 4],
    ... [5, 6]
    ... ]
    >>> print(matrix_mul(mtx_a, mtx_b))
    [[13, 16]]

The function also works with floating-point numbers.

::

    >>> mtx_a = [
    ... [1.2, 5.5, 6.2],
    ... [4.66, 12.3, -9.2]
    ... ]
    >>> mtx_b = [
    ... [5.0, 3.3],
    ... [-2.9, 4.4],
    ... [7.2, 4.4]
    ... ]
    >>> print(matrix_mul(mtx_a, mtx_b))
    [[34.69, 55.44000000000001], [-78.61, 29.018000000000008]]

Integers and floats can be combined.

::

    >>> mtx_a = [
    ... [1, 2.2, 3.3, 4],
    ... [5, 6, 7, 8.8],
    ... ]
    >>> mtx_b = [
    ... [1.1, 2, 3.3],
    ... [4.0, 5.5, 6],
    ... [7, 8, 9],
    ... [10.01, 11, 12.3]
    ... ]
    >>> print(matrix_mul(mtx_a, mtx_b))
    [[73.03999999999999, 84.5, 95.4], [166.58800000000002, 195.8, 223.74]]

A minimum of two arguments must be provided. Otherwise, a TypeError is raised.

::

    >>> print(matrix_mul()) # doctest: +NORMALIZE_WHITESPACE

ValueErrors
===========

If two matrices cannot be multiplied (ie. the row count of ``mtx_a`` is not
equal to the column count in ``mtx_b``), a ValueError is raised.

::

    >>> mtx_a = [
    ... [1, 2],
    ... [3, 4],
    ... ]
    >>> mtx_b = [
    ... [1, 2],
    ... [2, 3],
    ... [4, 5]
    ... ]
    >>> print(matrix_mul(mtx_a, mtx_b))
    Traceback (most recent call last):
    ValueError: mtx_a and mtx_b can't be multiplied


The parameters ``mtx_a`` and ``mtx_b`` cannot be empty. Otherwise, a ValueError
is raised.

::

    >>> print(matrix_mul([], [[1, 2]]))
    Traceback (most recent call last):
    ValueError: mtx_a can't be empty

::

    >>> print(matrix_mul([[1, 2]], [[]]))
    Traceback (most recent call last):
    ValueError: mtx_b can't be empty

::

    >>> print(matrix_mul([[]], []))
    Traceback (most recent call last):
    ValueError: mtx_a can't be empty

Invalid Matrices
================

The parameters ``mtx_a`` and ``mtx_b`` must be lists. If either parameter is
not a list, a TypeError is raised.

::

    >>> print(matrix_mul("not a list", [[1, 2]]))
    Traceback (most recent call last):
    TypeError: mtx_a must be a list

::

    >>> print(matrix_mul([[1, 2]], "also not a list"))
    Traceback (most recent call last):
    TypeError: mtx_b must be a list

::

    >>> print(matrix_mul("not a list", "also not a list"))
    Traceback (most recent call last):
    TypeError: mtx_a must be a list

::

    >>> print(matrix_mul(None, None))
    Traceback (most recent call last):
    TypeError: mtx_a must be a list

Not just any list - they *must* be lists of lists!

::

    >>> print(matrix_mul([1, 2], [[3, 4]]))
    Traceback (most recent call last):
    TypeError: mtx_a must be a list of lists

::

    >>> print(matrix_mul([[1, 2]], [3, 4]))
    Traceback (most recent call last):
    TypeError: mtx_b must be a list of lists

::

    >>> print(matrix_mul([1, 2], [3, 4]))
    Traceback (most recent call last):
    TypeError: mtx_a must be a list of lists

And not just any list of lists - they *must* be lists of lists containing
integers or floats!

::

    Traceback (most recent call last):
    TypeError: mtx_a should contain only integers or floats
